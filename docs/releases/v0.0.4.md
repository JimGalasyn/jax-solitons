# jax-solitons v0.0.4 — campaign robustness: FleetExecutor, self-healing REST, reap by label

> **Released 2026-06-16.** The campaign-robustness release. Everything from the 2026-06-15 farm post-mortem — a transient DNS storm that made the fleet treat **14 of 16 legs as terminally failed** and tore the boxes down — turned into hardening, plus the refactor that collapses three forked fleet drivers into one library executor. Closes all seven fleet-hardening issues (#23–#29). Entirely additive: no breaking changes from v0.0.3.

## Headline: `FleetExecutor` — a fleet run is data, not a fork

`jax_solitons.campaign.FleetExecutor` runs a list of `FleetLeg(label, command, ship, fetch)` over any `Provider`, one rented host per leg, in parallel:

```python
from jax_solitons.campaign import FleetExecutor, FleetLeg, SentinelReady, VastProvider

legs = [FleetLeg(label=f"eps_{e}",
                 command=f"python batch.py --eps {e} --out out",
                 ship=("batch.py",), fetch="out", done_when="out/manifest.json")
        for e in (0.1, 0.5, 1.0)]
FleetExecutor(VastProvider(), launch, local_out_dir="out", ready=SentinelReady(),
              max_parallel=12).run(legs)
```

The three hand-rolled fleet drivers that drove the 2026-06-15 campaign — each copy-pasting the same rent → wait-worker → scp → ssh → fetch → fail-over loop — now collapse to thin callers that just build legs. The executor is physics-agnostic (no model/stepper import); the only thing crossing the boundary is a shell command and the files it ships/fetches.

Every robustness feature is the fix for a specific failure that session paid for:

- **Per-leg failover** on a bad host (`HostProbeFailed`) or an offer race (the new `RentUnavailable` signal — the offer was taken before any instance was created, so there is nothing to tear down and the executor just tries the next offer).
- **Fast-fail a corpse** (`VastProvider.dead_reason`) — a container that never came up is detected via the API and failed over in seconds, instead of ssh-polling a dead box for the full ~20-minute readiness deadline.
- **Offer-pool refresh** when the pool drains under heavy failover, instead of `NO_OFFERS` while new offers exist.
- **Resume / skip** legs whose output marker already exists locally — idempotent relaunch (a re-launch after a crash at leg 12/16 redoes nothing already done).
- **Launch jitter** against the thundering herd, and a **signal-safe teardown** backstop that destroys in-flight rentals on SIGTERM/SIGINT before unwinding.

## Self-healing Vast REST (`_req`)

Every Vast API call now retries transient transport faults — DNS `EAI_AGAIN`, connection reset, read timeout, 5xx — with exponential backoff before giving up. This is the single change, at the one function every REST call flows through, that would have made the original DNS storm invisible.

The retry is **idempotent-aware**, which is a cost-safety invariant, not a nicety: `create` retries *only* a pre-send `EAI_AGAIN` (the server never saw the request), so a half-completed rent can never double-rent a GPU that bills by the second. Only `EAI_AGAIN` qualifies — a name that genuinely doesn't resolve (`EAI_NONAME`) is terminal and fails over at once. A raw `URLError` no longer escapes; every network failure surfaces as `VastError` (now carrying `.code`, so callers can tell a terminal 401/403 from a recoverable race).

## Orphan attribution: `LaunchSpec.label` + `reap --label`

Every rented instance is stamped with a campaign/run label at create (`LaunchSpec.label`), and `reap` gains a third scope:

```bash
python -m jax_solitons.campaign.reap --label eps-kick-farm --yes
```

`--label NAME` destroys only the instances stamped with that label. It is safe under concurrent farming like `--ledger` (no `--all` gate), but **ledger-free**, so a crashed run's orphans are reapable by campaign label from any machine. Scopes AND together (`--ledger` ∩ `--label`).

## Also new

- **`campaign status`** — `python -m jax_solitons.campaign.status --ledger <path>` prints the instances live right now (what is billing) plus cumulative spend and outcomes from the ledger, replacing log-grepping and shelling `vastai`.
- **`make_verlet_step`** is now exported from `jax_solitons.steppers` (was reachable only via the `.verlet` submodule).
